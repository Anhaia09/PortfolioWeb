Linguagem de Programação

A linguagem de programação é um método padronizado, formado por um conjunto de regras sintáticas e semânticas, de implementação de um código fonte
que pode ser compilado e transformado em um programa de computador, ou usado como script interpretado - que informará instruções de processamento ao computador.

Estruturas de decisão

IF/ELSE:
A estrutura de decisão if executa um bloco de instruções se o teste lógico for verdadeiro.
A estrutura de decisão if/else executa um bloco de instruções se o teste lógico for verdadeiro caso contrário um segundo bloco de instruções é executado.

SWITCH:
A estrutura de decisão switch permite que o fluxo do programa seja desviado para vários caminhos diferentes a depender do valor passado para análise

Estruturas de repetição:

WHILE:
A estrutura de repetição while permite a criação de um loop que executa um bloco de instruções repetidas vezes enquanto uma condição for verdadeira.
DO / WHILE:
Semelhante ao while o do/while obriga que o bloco de instruções seja executado ao menos uma vez.
FOR:
Possui a mesma ideia que o **while** porém com uma estrutura fixa que obriga a criação de uma variável de controle

Funções/Funções Recursivas

FUNÇÕES: são blocos de instruções que são executados somente quando “chamados”. Funções pode ter parâmetros, valores que são passados durante
sua chamada. Funções também podem ter retorno, um valor que é retornado para o local de chamada da função.

RECURSIVAS: são funções que executam a chamada delas mesmas dentro de seu próprio bloco de instruções.

Programação Orientada à Objeto

Estrutura de uma Classe: Classes em programação orientada a objetos são abstrações de objetos, ou seja, representam de forma genérica as instâncias dos objetos.
A estrutura de uma classe é divida nome da classe, atributos e métodos.

Encapsulamento

um dos princípios fundamentais da programação orientada a objetos, que visa ocultar os detalhes internos de uma classe e expor apenas as interfaces necessárias
para interagir com ela.
1 - public: Um membro (atributo ou método) marcado como público pode ser acessado de qualquer lugar, ou seja, de dentro da própria classe, de classes do mesmo pacote e de classes de pacotes diferentes.
2 - protected: Um membro protegido pode ser acessado apenas dentro da própria classe, de classes do mesmo pacote e de subclasses (mesmo que estejam em pacotes diferentes).
3 - private: Um membro privado só pode ser acessado dentro da própria classe. Ele não pode ser acessado de fora da classe, nem mesmo por classes do mesmo pacote ou subclasses.

Getters and Setters

Esses métodos permitem que os usuários da classe leiam e modifiquem os atributos de maneira controlada, garantindo a consistência e integridade dos dados.

Método Construtor

é um tipo especial de método que é chamado automaticamente quando uma instância de uma classe é criada. O objetivo principal de um método construtor é inicializar os atributos de um objeto, garantindo que ele esteja em um estado válido e consistente desde o momento de sua criação.

Diagrama UML

Existem vários tipos de diagramas UML, um dos mais utlizados é o diagrama de classes UML, que retrata um sistema estático, orientado a objetos. Ele define projetos por classes, atributos e funções; como tal, é um componente fundamental de qualquer solução orientada a objetos.
